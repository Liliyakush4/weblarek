# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и отображения данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах эндпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на эндпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.


#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.


# Данные
## Интерфейс IProduct
Интерфейс товара для использования во всех компонентах приложения.
Используется для отображения:
- Товаров в каталоге
- Детальной карточки товара
- Элементов корзины

Свойства:
`id: string` - уникальный идентификатор товара;
`description: string` - описание товара;
`image: string` - URL изображения товара;
`title: string` - название товара;
`category: string` - категория товара;
`price: number | null` - цена товара. Может быть null, если цена не установлена.


## Интерфейс IBuyer
Интерфейс данных покупателя для оформления заказа.
Используется на этапе оформления заказа для сбора информации.

Свойства:
`payment: 'card' | 'cash' | ''` - выбор способа оплаты;
`address: string` - указание адреса доставки;
`email: string` - адрес электронной почты для связи;
`phone: string` - номер телефона для связи.

## Модели данных
### Класс Products
Управляет каталогом товаров. Хранит массив всех товаров и товар, выбранный для подробного отображения.

Конструктор:
`constructor(initialProducts?: IProduct[])` - В конструктор передается начальный массив товаров для инициализации каталога.

Поля класса:
`products: IProduct[]` - массив всех товаров каталога.
`selectedCard: IProduct | null` - товар, выбранный для подробного отображения.

Методы класса:
`saveData(data: IProduct[]): void` - сохраняет массив товаров, полученный в параметрах метода.
`getProductList(): IProduct[]` - получает массив товаров из модели.
`getProduct(id: string): IProduct | undefined` - получает один товар по его id.
`saveCard(card: IProduct): void` - сохраняет товар для подробного отображения.
`getCard(): IProduct | null` - получает товар для подробного отображения.


### Класс ShoppingCart
Управляет корзиной товаров. Хранит массив товаров, выбранных покупателем для покупки. В корзине каждый товар может быть только в одном экземпляре.

Конструктор:
`constructor(initialItems?: IProduct[])` - принимает начальный массив товаров в корзине.

Поля класса:
`items: IProduct[]` - массив товаров в корзине.

Методы класса:
`getItems(): IProduct[]` - получает массив товаров, которые находятся в корзине.
`addItem(product: IProduct): void` - добавляет товар, который был получен в параметре, в массив корзины.
`removeItem(productId: string): void` - удаляет товар, полученный в параметре из массива корзины.
`clear(): void` - очистка корзины.
`getTotalPrice(): number` - получает стоимость всех товаров в корзине.
`getTotalCount(): number` - получает длину массива товаров.
`checkProductInCart(productId: string): boolean` - проверяет наличие товара в корзине по его id, полученного в параметр метода.

### Класс BuyerData
Управляет данными покупателя. Хранит информацию о покупателе для оформления заказа.

Конструктор принимает начальные данные покупателя:
`constructor(initialData?: IBuyer)`

Поля класса:
`data: IBuyer` - данные покупателя.

Методы класса:
`setData(data: Partial<IBuyer>): void` - сохраняет данные в модели.
`getData(): IBuyer` - получает все данные покупателя.
`clear(): void` - очистка данных покупателя.
`validate(): { payment?: string; address?: string; email?: string; phone?: string }` — возвращает объект, где ключ — имя поля с ошибкой, значение — текст ошибки. Отсутствие ключа = поле валидно.

## Слой коммуникации
### Класс WebLarekApi
Класс коммуникационного слоя. Использует композицию с базовым Api для HTTP-запросов.

Конструктор:
`constructor(api: IApi)` - принимает объект, реализующий интерфейс `IApi`.

Методы класса:
`getProductList(): Promise<IProduct[]>` — GET `/product/`, возвращает массив `IProduct[]`.
`submitOrder(orderData: OrderData): Promise<OrderResult)` — POST `/order/` с данными заказа, возвращает результат.


# Представление
## Интерфейс IHeader
Интерфейс шапки приложения.

Свойства:
`counter: number` - количество товаров в корзине для отображения в счетчике.

## Интерфейс IModal
Базовый интерфейс для всех модальных окон в приложении.

Методы:
`show(): void` - показывает модальное окно;
`hide(): void` - скрывает модальное окно;
`readonly isOpen: boolean` - проверяет состояние видимости.

Свойства:
`onClose?: () => void` - опциональный колбэк при закрытии модального окна.

## Интерфейс IActionModal
Интерфейс для модальных окон с кнопками действий. Наследует `IModal`.

Свойства:
`buttonText: string` - текст основной кнопки действия;
`isButtonDisabled?: boolean` - состояние активности кнопки;
`onAction?: () => void` - колбэк при нажатии на кнопку;

Методы:
`setButtonDisabled(disabled: boolean): void` - устанавливает состояние кнопки;
`setButtonText(text: string): void` - устанавливает текст кнопки.

## Интерфейс IContentModal
Интерфейс для модальных окон с динамическим контентом. Наследует `IModal`.

Свойства:
`content: HTMLElement | string` - содержимое модального окна.

Методы:
`setContent(content: HTMLElement | string): void` - устанавливает содержимое элемента.

## Интерфейс IProductCardModal
Интерфейс описывает структуру данных товара для отображения в модальном окне. Используется для передачи информации о товаре между слоями приложения.

Свойства:
`id: string` - ID товара для идентификации товара в системе, связывания с сервером и управления состоянием в корзине;
`description: string` - описание товара. Предоставляет пользователю детальную информацию о товаре;
`image: string` - URL-адрес изображения товара;
`title: string` - название товара и главный заголовок карточки;
`category: string` - категория товара. Определяет визуальное оформление через CSS-классы и группирует товары по типам;
`price: number | null` - цена в синапсах. number - отображается как "X синапсов", а null - отображается как "Бесценно" (для товаров без цены).

## Интерфейс IProductCardModalView 
Интерфейс описывает поведение и возможности компонента модального окна карточки товара. Наследует базовый интерфейс модальных окон и добавляет специфичные для карточки товара методы.

Методы:
`setProductData(data: IProductCardModal): void` - метод для обновления данных карточки товара. Принимает объект с данными товара;
`setAddToCardHandler(handler: (product: IProductCardModal) => void): void` - устанавливает обработчик добавления товара в корзину;
`setInCardStatus(inCart: boolean): void` - визуально обновляет статус состояния кнопки добавления товара в корзину. `inCart: boolean` - флаг наличия товара в корзине;
`onAddToCard?: (product: IProductCardModal) => void` - колбек-событие добавления товара в корзину.

## Интерфейс ISuccessModal
Специализированный интерфейс для модального окна успешного заказа. Наследует `IModal`.

Методы:
`setTotal(value: number): void` - устанавливает сумму заказа.

Свойства:
`onAction?: () => void` - колбэк при действии (нажатии на кнопку).

## Интерфейс SuccessModalData
Интерфейс данных для модального окна успешного заказа.

Свойства:
`total: number` - сумма покупки;
`image?: string` - опциональный путь к изображению успешного заказа.

Класс реализует шапку приложения, которая содержит счетчик товаров в корзине и кнопку для открытия корзины. Наследуется от базового класса `Component`, дженериком которого является интерфейс`IHeader`, и реализует реактивное обновление интерфейса.

Поля класса:
`protected counterElement: HTMLElement` - ссылка на DOM-элемент счетчика товаров в корзине;
`protected basketButton: HTMLButtonElement` - ссылка на кнопку открытия корзины;

Конструктор:
`constructor(protected events: IEvents, container: HTMLElement)` - находит и сохраняет ссылку на DOM-элемент с помощью метода `ensureElement`, устанавливает обработчик события на кнопку корзины.

Методы:
`set counter(value: number): void` - обновляет отображаемое количество товаров вкорзине;

### Класс ProductCardModal
Класс компонента модального окна карточки товара. Отвечает за отображение детальной информации о товаре и взаимодействие с пользователем. Управляет состоянием кнопки добавления в корзину.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - находит и сохраняет ссылки на все DOM-элементы карточки, устанавливает обработчики событий.

Методы:
`show(): void`- открывает модальное окно и генерирует событие `productModal:open`;

`get isOpen(): boolean` - геттер - возвращает true, если модальное окно открыто;
`setProductData(data: IProductCardModal): void` - обновляет все данные карточки товара единым методом;
`setAddToCardHandler(handler: (product: IProductCardModal) => void): void` - устанавливает обработчик события добавления товара в корзину;
`setInCardStatus(inCart: boolean): void` - обновляет состояние кнопки "В корзину";
`set id(value: string)` - уохраняет ID товара в data-атрибут productId контейнера;
`set image(src: string)` - устанавливает источник изображения товара;
`set title(value: string)` - устанавливает заголовок товара;
`set description(value: string)` - устанавливает описание товара;
`set category(value: string)` - устанавливает категорию товара;
`set price(value: number | null)` - устанавливает цену товара: null - отображает "Бесценно", number - отображает "{value} синапсов";
`protected bindEvents(): void` - инициализирует обработчики событий: клик по кнопке "В корзину" - `handleAddToCard()`, клик на оверлей модального окна - `handleClose()`;
`protected handleAddToCard(): void` - обрабатывает добавление товара в корзину: собирает текущие данные из DOM-элементов, создает объект `IProductCardModal`, генерирует событие `product:addToCard`;
`protected handleClose(): void` - Обрабатывает закрытие модального окна через hide() и генерирует событие modal:close.
`protected updateCategoryStyles(category: string): void` - Обновляет CSS-классы категории товара: удаляет все существующие классы категорий, добавляет соответствующий класс через `getCategoryClass()`;
`protected getCategoryClass(category: string): string` - Возвращает CSS-класс для категории товара;
`protected getCategoryBemClass(category: string): { name: string, class: string }` - извлекает числовое значение цены из текстового содержимого: "Бесценно" - null, "100 синапсов" - 100;
`protected setText(element: HTMLElement, text: string): void` - утилитарный метод для установки текстового содержимого элемента;
`protected createActionButton(text: string, disabled?: boolean): HTMLButtonElement` - cоздает кнопку действия с заданными параметрами.

### Класс SuccessModal
Класс модального окна успешного оформления заказа. Отображает информацию о завершенном заказе с суммой покупки. Наследуется от `Component<SuccessModalData>`. Реализует интерфейс `ISuccessModal`.

Поля класса:
`private descriptionElement: HTMLElement = null!`- приватное поле для хранения ссылки на DOM-элемент с описанием заказа. Отображает текст "Списано X синапсов";
`private closeButton: HTMLButtonElement = null!` - приватное поле для хранения ссылки на кнопку закрытия модального окна с текстом "За новыми покупками!";
`private imageElement: HTMLImageElement = null!` - приватное поле для хранения ссылки на динамически создаваемый элемент изображения, отображающий визуальное подтверждение успешного заказа;
`private modalContainer: HTMLElement = null!` - приватное поле для хранения ссылки на корневой контейнер модального окна;
`public onClose?: () => void` - публичный опциональный колбэк, вызываемый при закрытии модального окна. Используется для выполнения дополнительных действий после закрытия;
`public onAction?: () => void` - публичный опциональный колбэк, вызываемый при нажатии на основную кнопку действия. Обеспечивает реакцию на пользовательское взаимодействие.

Конструктор:
`constructor(container: HTMLElement)` - инициализирует родительский класс Component, находит корневой контейнер модального окна, выполняет инициализацию DOM-элементов, создает элемент изображения, устанавливает обработчики событий.

Методы класса:
`private initializeElements(): void` - приватный метод для поиска и инициализации DOM-элементов внутри контейнера модального окна;
`private createImageElement(): void` - приватный метод для динамического создания и размещения элемента изображения в структуре модального окна;
`private bindEvents(): void` - приватный метод для установки обработчиков событий на элементы модального окна;
`set total(value: number): void` - cеттер для установки общей суммы заказа. Автоматически вызывается методом `render()` при обновлении данных.
`set image(src: string): void` - cеттер для установки пути к изображению успешного заказа. Использует метод `setImage` из базового класса `Component`.
Сохраняет альтернативный текст для доступности;
`setTotal(value: number): void` - публичный метод интерфейса `ISuccessModal` для установки суммы заказа. Является оберткой вокруг сеттера `total`;
`show(): void` - отображает модальное окно;
`hide(): void` - скрывает модальное окно;
`get isOpen(): boolean` - геттер для проверки текущего состояния видимости модального окна;
`private handleClose(): void` - приватный метод для обработки события закрытия модального окна;
