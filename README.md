# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и отображения данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах эндпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на эндпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.


#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.


# Данные
## Интерфейс IProduct
Интерфейс товара для использования во всех компонентах приложения.
Используется для отображения:
- Товаров в каталоге
- Детальной карточки товара
- Элементов корзины

Свойства:
`id: string` - уникальный идентификатор товара;
`description: string` - описание товара;
`image: string` - URL изображения товара;
`title: string` - название товара;
`category: string` - категория товара;
`price: number | null` - цена товара. Может быть null, если цена не установлена.


## Интерфейс IBuyer
Интерфейс данных покупателя для оформления заказа.
Используется на этапе оформления заказа для сбора информации.

Свойства:
`payment: 'card' | 'cash' | ''` - выбор способа оплаты;
`address: string` - указание адреса доставки;
`email: string` - адрес электронной почты для связи;
`phone: string` - номер телефона для связи.

## Интерфейс IProductsModel
Интерфейс для работы с данными каталога товаров. 
Определяет контракт для модели, управляющей каталогом товаров приложения.

Методы:
`saveData(products: IProduct[]): void` - сохраняет массив товаров в модель;
`getProductList(): IProduct[]` - возвращает полный список товаров из каталога;
`getProduct(id: string): IProduct | undefined` - находит и возвращает товар по его уникальному идентификатору.

## Интерфейс ICartModel
Интерфейс для работы с корзиной покупок. Определяет контракт для модели, управляющей состоянием корзины и операциями с товарами в ней.

Методы:
`addItem(product: IProduct): void` - добавляет товар в корзину;
`removeItem(id: string): void` - удаляет товар из корзины по его идентификатору;
`clear(): void` - полностью очищает корзину;
`getItems(): IProduct[]` - возвращает массив товаров, находящихся в корзине;
`getTotalCount(): number` - возвращает общее количество товаров в корзине.

## Интерфейс IBuyerModel
Интерфейс для работы с данными покупателя.
Определяет контракт для модели, управляющей информацией о покупателе в процессе оформления заказа.

Методы:
`setData(data: Partial<IBuyer>): void` - устанавливает или обновляет данные покупателя;
`getData(): IBuyer` - возвращает полные данные покупателя;
`clear(): void` - очищает все данные покупателя;
`validate(): BuyerValidationErrors` - выполняет валидацию данных покупателя и возвращает объект с ошибками.

## Модели данных
### Класс Products
Управляет каталогом товаров. Хранит массив всех товаров и товар, выбранный для подробного отображения.

Конструктор:
`constructor(private events: IEvents, initialData?: IProduct[])` - принимает брокер событий и начальный массив товаров для инициализации каталога.

Поля класса:
`_products: IProduct[]` - массив всех товаров каталога. Доступ к нему осуществляется через геттер `products` и метод `getProductList()`, которые возвращают копию массива.
`_selectedCard: IProduct | null` - товар, выбранный для подробного отображения. Доступ к нему осуществляется через геттер `selectedCard` и метод `getCard()`, которые возвращают копию объекта.

Методы класса:
`saveData(data: IProduct[]): void` - сохраняет массив товаров, полученный в параметрах метода.
`getProductList(): IProduct[]` - получает массив товаров из модели.
`getProduct(id: string): IProduct | undefined` - получает один товар по его id.
`saveCard(card: IProduct): void` - сохраняет товар для подробного отображения.
`getCard(): IProduct | null` - получает товар для подробного отображения.


### Класс ShoppingCart
Управляет корзиной товаров. Хранит массив товаров, выбранных покупателем для покупки. В корзине каждый товар может быть только в одном экземпляре.

Конструктор:
`constructor(private events: IEvents, initialItems?: IProduct[])` - принимает брокер событий и начальный массив товаров в корзине.

Поля класса:
`items: IProduct[]` - массив товаров в корзине.

Методы класса:
`getItems(): IProduct[]` - получает массив товаров, которые находятся в корзине.
`addItem(product: IProduct): void` - добавляет товар, который был получен в параметре, в массив корзины.
`removeItem(productId: string): void` - удаляет товар, полученный в параметре из массива корзины.
`clear(): void` - очистка корзины.
`getTotalPrice(): number` - получает стоимость всех товаров в корзине.
`getTotalCount(): number` - получает длину массива товаров.
`checkProductInCart(productId: string): boolean` - проверяет наличие товара в корзине по его id, полученного в параметр метода.

### Класс BuyerData
Управляет данными покупателя. Хранит информацию о покупателе для оформления заказа.

Конструктор:
`constructor(private events: IEvents, initialData?: IBuyer)` - принимает брокер событий и начальные данные покупателя.

Поля класса:
`data: IBuyer` - данные покупателя.

Методы класса:
`setData(data: Partial<IBuyer>): void` - сохраняет данные в модели;
`getData(): IBuyer` - получает все данные покупателя;
`clear(): void` - очистка данных покупателя;
`validate(): BuyerValidationErrors` — возвращает объект, где ключ — имя поля с ошибкой `(payment, address, email, phone)`, а значение — текст ошибки. Отсутствие ключа означает, что поле валидно. Тип `BuyerValidationErrors` описан в types и соответствует структуре `{ payment?: string; address?: string; email?: string; phone?: string }`.

## Слой коммуникации
### Класс WebLarekApi
Класс коммуникационного слоя. Использует композицию с базовым Api для HTTP-запросов.

Конструктор:
`constructor(api: IApi)` - принимает объект, реализующий интерфейс `IApi`.

Методы класса:
`getProductList(): Promise<IProduct[]>`— выполняет GET-запрос на эндпоинт `/product/` и возвращает промис с массивом товаров `IProduct[]`, полученным от сервера.
Жобавление CDN-префикса выполняется на уровне слоя представления в абстрактном классе `CardWithCategoryImage`, который формирует полный URL на основе имени файла и константы `CDN_URL`.
`submitOrder(orderData: OrderData): Promise<OrderResult)` — POST-запрос `/order/` с данными заказа. Принимает объект с данными заказа. Возвращает результат оформления заказа `OrderResult`.

### Абстрактный Класс Card
Базовый класс для всех типов карточек товара в приложении. Предоставляет общую функциональность для отображения основной информации о товаре.

Поля класса:

Методы класса:
`set title(value: string): void` - устанавливает название товара;
`set price(value: number | null): void` - устанавливает цену товара;
`set id(value: string): void` — сохраняет идентификатор товара во внутреннем приватном поле;
`get id(): string` — возвращает сохранённый идентификатор товара.

### Абстрактный Класс Form
Базовый класс для всех форм в приложении. Инкапсулирует общую логику работы с формами: обработку отправки, управление ошибками, состояние кнопок и эмит событий изменений.

Конструктор принимает брокер событий для коммуникации между компонентами и DOM-элемент формы: `constructor(protected events: IEvents, container: HTMLElement)`.

Абстрактные свойства:
`protected abstract formName: string` - должно быть переопределено в дочерних классах для идентификации формы при эмите событий.

Поля класса:
`submitButton: HTMLButtonElement` - кнопка отправки формы;
`errorsElement: HTMLElement` - контейнер для отображения ошибок.

Методы класса:
`protected handleSubmit(evt: Event): void` - обработчик события submit формы. Предотвращает стандартное поведение и вызывает submitForm() если кнопка не заблокирована;
`protected abstract submitForm(): void`
Абстрактный метод, должен быть реализован в дочерних классах. Содержит логику отправки конкретной формы;
`protected emitFormChange(data: object): void` - универсальный метод для эмита событий изменений в форме;
`enableSubmitButton(): void` - активирует кнопку отправки формы;
`disableSubmitButton(): void` - деактивирует кнопку отправки формы.

# Презентер
Презентер является центральным координатором приложения, связывающим модели данных с представлениями через систему событий. В проекте один презентер. Он размещен в файле main.ts, поскольку приложение состоит из одной страницы и весь функционал организован в соответствующих модальных окнах.

Функции, которые выполняет презентер:
- создает основные компоненты приложения;
- обрабатывает события моделей: для каталога товаров, выбора товара и изменения корзины;
- обрабатывает события представления для взаимодействия с товарами, работой с корзиной, - оформления заказа, синхронизации данных форм;
- управляет состоянием приложения для текущее модальное окно и загрузку данных при запуске.

События в моделях данных:
`catalog:changed` - изменение каталога товаров;
`product:selected` - изменение выбранного товара;
`cart:changed` - изменение состава корзины и счетчика;
`buyer:changed` - изменение данных покупателя.

События в слое представления:
`product:open` - выбор карточки для просмотра;
`card:action` - нажатие кнопки покупки/удаления товара;
`cart:remove` - нажатие кнопки удаления товара из корзины;
`basket:open` - нажатие кнопки открытия корзины;
`order:open` - нажатие кнопки оформления заказа;
`order:step1:submit` - нажатие кнопки перехода ко второй форме оформления заказа;
`order:step2:submit` - нажатие кнопки оплаты/завершения оформления заказа;
`success:close` - закрытие окна успешного заказа;
`form:change` - изменение данных в формах;
`modal:closed` - закрытие модального окна.
