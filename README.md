# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и отображения данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах эндпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на эндпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.


#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.


# Данные
## Интерфейс IProduct
Интерфейс товара для использования во всех компонентах приложения.
Используется для отображения:
- Товаров в каталоге
- Детальной карточки товара
- Элементов корзины

Свойства:
`id: string` - уникальный идентификатор товара;
`description: string` - описание товара;
`image: string` - URL изображения товара;
`title: string` - название товара;
`category: string` - категория товара;
`price: number | null` - цена товара. Может быть null, если цена не установлена.


## Интерфейс IBuyer
Интерфейс данных покупателя для оформления заказа.
Используется на этапе оформления заказа для сбора информации.

Свойства:
`payment: 'card' | 'cash' | ''` - выбор способа оплаты;
`address: string` - указание адреса доставки;
`email: string` - адрес электронной почты для связи;
`phone: string` - номер телефона для связи.

## Интерфейс IProductsModel
Интерфейс для работы с данными каталога товаров. 
Определяет контракт для модели, управляющей каталогом товаров приложения.

Методы:
`saveData(products: IProduct[]): void` - сохраняет массив товаров в модель;
`getProductList(): IProduct[]` - возвращает полный список товаров из каталога;
`getProduct(id: string): IProduct | undefined` - находит и возвращает товар по его уникальному идентификатору.

## Интерфейс ICartModel
Интерфейс для работы с корзиной покупок. Определяет контракт для модели, управляющей состоянием корзины и операциями с товарами в ней.

Методы:
`addItem(product: IProduct): void` - добавляет товар в корзину;
`removeItem(id: string): void` - удаляет товар из корзины по его идентификатору;
`clear(): void` - полностью очищает корзину;
`getItems(): IProduct[]` - возвращает массив товаров, находящихся в корзине;
`getTotalCount(): number` - возвращает общее количество товаров в корзине.

## Интерфейс IBuyerModel
Интерфейс для работы с данными покупателя.
Определяет контракт для модели, управляющей информацией о покупателе в процессе оформления заказа.

Методы:
`setData(data: Partial<IBuyer>): void` - устанавливает или обновляет данные покупателя;
`getData(): IBuyer` - возвращает полные данные покупателя;
`clear(): void` - очищает все данные покупателя;
`validate(): BuyerValidationErrors` - выполняет валидацию данных покупателя и возвращает объект с ошибками.

## Модели данных
### Класс Products
Управляет каталогом товаров. Хранит массив всех товаров и товар, выбранный для подробного отображения.

Конструктор:
`constructor(initialProducts?: IProduct[])` - В конструктор передается начальный массив товаров для инициализации каталога.

Поля класса:
`products: IProduct[]` - массив всех товаров каталога.
`selectedCard: IProduct | null` - товар, выбранный для подробного отображения.

Методы класса:
`saveData(data: IProduct[]): void` - сохраняет массив товаров, полученный в параметрах метода.
`getProductList(): IProduct[]` - получает массив товаров из модели.
`getProduct(id: string): IProduct | undefined` - получает один товар по его id.
`saveCard(card: IProduct): void` - сохраняет товар для подробного отображения.
`getCard(): IProduct | null` - получает товар для подробного отображения.


### Класс ShoppingCart
Управляет корзиной товаров. Хранит массив товаров, выбранных покупателем для покупки. В корзине каждый товар может быть только в одном экземпляре.

Конструктор:
`constructor(initialItems?: IProduct[])` - принимает начальный массив товаров в корзине.

Поля класса:
`items: IProduct[]` - массив товаров в корзине.

Методы класса:
`getItems(): IProduct[]` - получает массив товаров, которые находятся в корзине.
`addItem(product: IProduct): void` - добавляет товар, который был получен в параметре, в массив корзины.
`removeItem(productId: string): void` - удаляет товар, полученный в параметре из массива корзины.
`clear(): void` - очистка корзины.
`getTotalPrice(): number` - получает стоимость всех товаров в корзине.
`getTotalCount(): number` - получает длину массива товаров.
`checkProductInCart(productId: string): boolean` - проверяет наличие товара в корзине по его id, полученного в параметр метода.

### Класс BuyerData
Управляет данными покупателя. Хранит информацию о покупателе для оформления заказа.

Конструктор принимает начальные данные покупателя:
`constructor(initialData?: IBuyer)`

Поля класса:
`data: IBuyer` - данные покупателя.

Методы класса:
`setData(data: Partial<IBuyer>): void` - сохраняет данные в модели.
`getData(): IBuyer` - получает все данные покупателя.
`clear(): void` - очистка данных покупателя.
`validate(): { payment?: string; address?: string; email?: string; phone?: string }` — возвращает объект, где ключ — имя поля с ошибкой, значение — текст ошибки. Отсутствие ключа = поле валидно.

## Слой коммуникации
### Класс WebLarekApi
Класс коммуникационного слоя. Использует композицию с базовым Api для HTTP-запросов.

Конструктор:
`constructor(api: IApi)` - принимает объект, реализующий интерфейс `IApi`.

Методы класса:
`getProductList(): Promise<IProduct[]>` — GET-запрос `/product/`. Возвращает массив `IProduct[]`с обработанными URL изображений: преобразует расширение .svg в .png; формирует полный CDN-URL для изображений, используя базовый URL из константы CDN_URL; сохраняет относительные пути для изображений, которые уже являются абсолютными URL.
`submitOrder(orderData: OrderData): Promise<OrderResult)` — POST-запрос `/order/` с данными заказа. Принимает объект с данными заказа. Возвращает результат оформления заказа `OrderResult`

# Представление
## Интерфейсы слоя представления
Эти интерфейсы определяют структуру данных, которые передаются в метод render() соответствующих компонентов представления. Каждый интерфейс соответствует конкретному UI-компоненту.

Интерфейс для отображения шапки приложения.

Свойства:
`counter: number` - количество товаров в корзине для отображения в счетчике.

Интерфейс для отображения галереи товаров.

Свойства:
`items: IProduct[]` - массив товаров для отображения в каталоге.

## Интерфейс IModal
Интерфейс управления состоянием модального окна.

Свойства:
`isOpen: boolean` - флаг открытия/закрытия модального окна
`content?: HTMLElement | null` - DOM-элемент для отображения в качестве содержимого

Интерфейс для отображения детальной информации о товаре.

Свойства:
`product: IProduct` - данные товара;
`inCart: boolean` - флаг нахождения товара в корзине.

## Интерфейс ICatalogCard
Интерфейс для отображения карточки товара в каталоге.

Свойства:
`id: string` - уникальный идентификатор товара;
`title: string` - название товара;
`category: string` - категория товара;
`priceText: string` - текстовое представление цены;
`image: string` - URL изображения товара;
`disabled?: boolean` - флаг неактивного состояния карточки.

Свойства:

# Контроллер
Контроллер является центральным координатором приложения, связывающим модели и представления через систему событий. Он инициализирует слои моделей, API и представления.

Обработчики событий
Навигация и просмотр:
`basket:open` - открытие корзины;
`product:open` - открытие карточки товара;
`modal:close` - закрытие модальных окон.

Управление корзиной:
`product:add` - добавление товара в корзину;
`product:remove` - удаление товара из карточки;
`cart:remove` - удаление товара из корзины.

Оформление заказа:
`order:open` - начало оформления заказа;
`order:step1:submit` - отправка первого шага (адрес и оплата);
`order:step2:submit` - отправка второго шага (контакты) и финальная обработка заказа.

Вспомогательные функции:
`loadCatalog()` - загрузка каталога товаров с fallback на локальные данные;
`renderCatalog()` - отрисовка каталога товаров;
`openBasket()` - открытие модального окна корзины;
`updateBasketCounter()` - обновление счетчика товаров в шапке;
`calculateTotal()` - вычисление общей суммы заказа;
`isProductInCart()` - проверка наличия товара в корзине.
